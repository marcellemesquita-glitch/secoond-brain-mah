# Regras de Uso do Segundo Cérebro

## Papel de cada ferramenta

| Local                    | Finalidade                                      | Responsável     |
|--------------------------|-------------------------------------------------|-----------------|
| Memória da Judith        | Contexto operacional, preferências, fatos atuais | Judith          |
| Segundo Cérebro (este repo) | Conhecimento estruturado, projetos, áreas     | Judith + Mah    |
| Inbox                    | Capturas rápidas ainda não organizadas          | Judith organiza |

## Autonomia da Judith

- Judith pode **criar novas notas, pastas e áreas** quando identificar que algo relevante surgiu e ainda não existe lugar adequado
- Exemplos: novo projeto, nova área da vida, nova pessoa importante, novo recurso
- Sempre que possível, seguir o padrão de pastas existente
- Se a informação ainda estiver em fase de captura, salvar primeiro na `inbox/`

## Versionamento automático

- Judith deve fazer **commit + push** sempre que criar ou atualizar notas relevantes
- Usar mensagens claras (ex: "docs: adicionar nota sobre X")
- Objetivo: manter o repositório sempre atualizado no GitHub

## Quando preencher a Inbox

Judith deve salvar na `inbox/` quando:

- A informação for relevante para médio/longo prazo
- Ainda não existir uma nota específica sobre o tema
- Vale a pena guardar mesmo depois que a tarefa for feita

Judith **não** deve salvar quando:
- É uma tarefa executável simples (vai para Todoist/ClickUp)
- É algo muito passageiro ou irrelevante depois
- Já existe uma nota adequada

## Gatilhos de captura

- Durante conversas naturais
- Ao identificar contexto relevante em planejamentos
- Durante revisões semanais (Friday Review / Sunday Planning)
- Quando Mah solicitar explicitamente

## Organização da Inbox

- Revisão sugerida: **Sexta-feira** (durante Weekly Review)
- Organização: mover conteúdo relevante para `projetos/`, `areas/`, `pessoas/` ou `recursos/`

## Atualização

Judith pode criar e atualizar notas livremente quando identificar informação relevante.
